Visión general

Library Database in Spanish es un programa que sirve como servidor de una app que controla los préstamos, contenidos y usuarios de una biblioteca, teniendo funcionalidades para usuarios anónimos, clientes de la biblioteca, trabajadores de la biblioteca y administradores del sistema.

Note
 Se recomienda leer el Empezando del README.md antes de ejecutar el programa.

Cómo ejecutar la aplicación

  1. En src/main/resources/application.properties, cambia el valor de jwt.issuer a tu compañía, jwt.secret a una string de al menos 256 bits.

  2. Abre una terminal en el directorio del proyecto.

  3. Ejecuta el siguiente comando para ejecutar la aplicación:

    ./mvnw spring-boot:run

El usuario no tiene autorización para realizar la solicitud, pero puede conseguirla

El usuario no tiene los permisos suficientes para realizar la solicitud

El recurso que se pide no ha sido encontrado

El token de refresco ha caducado

El servidor no ha realizado la solicitud (usado para los errores que no son seguidos en el programa)

Autorización y autenticación

Library Database in Spanish utiliza tokens para la autorización de solicitudes. Aun así, hay partes de la API que sí son accesibles sin autorización

Pares access/refresh token

La autorización de la API se hace a través de pares de access y refresh tokens

Son algo así:

Unresolved directive in index.adoc - include::D:\Programación\Java\database-server\target/generated-snippets/registrarse/http-response.adoc[]

Access token

Los access token son JWT que contienen información sobre el usuario y sirven para autorizar las solicitudes.

En cada petición hay que añadir un header Authorization con la siguiente estructura:

Authorization: Bearer {access-token}

Solo tienen validez por cinco minutos, por lo que deben ser refrescados y no necesitan ser guardados con gran seguridad

Refresh token

Los refresh token son tokens guardados en base de datos que sirven para conseguir un nuevo access token, por lo que su validez es mayor (una semana) y debe ser guardado con seguridad por ello.

Para aprender a usarlos mire Cómo conseguir un nuevo token una vez ha caducado

Cómo conseguir un nuevo token una vez ha caducado

Cuando el access token ha caducado, hay que enviar una petición GET a /auth/token/refresh/{token} donde {token} es el refresh token entregado al registrarse.

Esta petición devuelve un par access/refresh token para tener autorización a los servicios de la API. Habrá que cambiar los actuales access y refresh tokens a los devueltos por esta petición, ya que los antiguos se eliminan.

Si el refresh token ha caducado también haga lo indicado en el siguiente apartado.
